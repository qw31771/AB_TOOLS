# CLAUDE.md - AB Tools 项目维护手册

## 1. 项目愿景与目标
* **定位**：Maya 通用工具集成插件。
* **核心目的**：
    * **环境自动化**：通过拖放安装，自动处理 Maya 环境变量（`sys.path`），无需手动配置。
    * **工具集成化**：为外部或自定义脚本提供统一的 UI 管理入口，实现快速启动。
    * **开发高效化**：支持开发者快速将新编写的脚本注册到插件 UI 中，降低工具分发和维护成本。

## 2. 核心架构与逻辑
项目遵循 **“安装 -> 引导 -> 逻辑 -> 表现”** 的解耦架构。

* **安装逻辑 (`install.py`)**：利用 `onMayaDroppedPythonFile` 回调。负责获取当前路径、注入 `sys.path` 并执行 `ab_tools.startup()`。
* **引导逻辑 (`ab_tools/__init__.py`)**：负责模块重载（Reload）策略，确保开发阶段代码修改能即时生效，并调用 `main.py` 的初始化函数。
* **管理逻辑 (`ab_tools/main.py`)**：
    * **自启动逻辑**：在 Maya 顶层控制栏创建 `AB_TOOLS` 工具架。
    * **交互逻辑**：创建 `shelfButton`（包含 `base` 和 `hover` 双态图标）及隔离符。
    * **路由逻辑**：按钮触发后，调用 `ui/` 目录下的接口，不直接包含 UI 布局代码。
* **UI 表现层 (`ab_tools/ui/`)**：基于 PySide2 和 `MayaQWidgetDockableMixin` 实现可停靠的窗口。

## 3. 开发规范 (AI 与开发者必读)

### ⚠️ 目录与文件约束 (严格准则)
1.  **禁止修改目录结构**：必须保持现有的 `AB_TOOLS/` 根目录及子目录层级。
2.  **增量原则**：允许增加新文件（如在 `tools/` 下增加新脚本），**严禁减少或删除**现有框架文件。
3.  **命名保护**：严禁对现有文件、类名、核心函数名进行重命名。
4.  **模块化要求**：脚本逻辑必须解耦。UI 逻辑严禁出现在 `main.py` 中；核心配置严禁硬编码在 `ui/` 脚本中。


##  操作指南
1.  **安装**：将整个 `AB_TOOLS` 文件夹存放在任意位置，拖拽 `install.py` 到 Maya 视图窗口。
2.  **启动**：点击工具架 `AB_TOOLS` 上的图标。
3.  **开发**：在 `ui/` 目录下编写新界面，在 `main.py` 中注册对应的启动按钮。

### 脚本格式规范
* **编码**：统一使用 `# -*- coding: utf-8 -*-`。
* **环境**：Python 3 (Maya 2022+)。
* **UI 库**：仅限使用 `PySide2`（保持与 Maya 原生库兼容）。
* **路径处理**：使用 `os.path` 进行动态路径获取，禁止使用绝对路径。
* **配置分层规范**：
    * **config_data.py**：仅存放物理路径计算、Maya API 原始值读取、基础数值定义。
    * **__init__.py (Config 类)**：负责将数据组装成逻辑对象。严禁在业务代码中直接 import config_data，必须通过 Config 类访问。
### 配置管理双层准则 (Strict Separation)
1. **数值修改准则**：所有涉及具体的像素数值（如按钮宽、窗口高、边距值）必须写在 `对应文件/__init__.py` 中。
2. **逻辑修改准则**：涉及路径拼接算法、DPI 获取方式、数学计算公式的修改，必须在 ``对应文件/对应文件_data.py`
**严禁**在 逻辑文件中硬编码业务数值。
